# Cinder Security Scan — 2026-04-02

## Summary
2 issues fixed, 2 low-severity notes.

---

## [FIXED — HIGH] Command Injection in GitService.swift

**File**: `Sources/Cinder/Services/GitService.swift`
**Pattern**: `shell("git -C \"\(url.path)\" ...")`

The `shell()` function passed `url.path` directly into a `/bin/zsh -c` command string. A directory named `/Local_Dev/projects/evil"; rm -rf ~/` would execute arbitrary shell commands.

**Fix**: Replaced `shell()` with a `git(_ args:, at:)` helper that calls `Process` with array arguments — git is called directly, no shell interpolation.

---

## [FIXED — MEDIUM] AppleScript Injection in ClaudeSkillsService.swift

**File**: `Sources/Cinder/Services/ClaudeSkillsService.swift`
**Pattern**: `do script "cd \"\(path)\" && claude ..."` in NSAppleScript

Path was interpolated raw into the AppleScript `do script` command. A project path containing `"` or `&&` followed by arbitrary commands would execute in Terminal.

**Fix**: Escape `"` → `\"` and `\` → `\\` in path before interpolation. `skill.invocation` is a static enum constant — safe.

---

## [LOW] API Key in UserDefaults

**File**: `Sources/Cinder/Services/CinderAPIServer.swift`
**Detail**: API key stored in `UserDefaults.standard`. On a non-sandboxed macOS app, UserDefaults plists are readable by any process running as the same user.

**Recommendation**: Migrate to `SecItemAdd`/`SecItemCopyMatching` (Keychain). Acceptable for personal local tool; upgrade before any multi-user deployment.

---

## [LOW] API Key Exposed in URL Query Param

**File**: `Sources/Cinder/Services/CinderAPIServer.swift`
**Detail**: `?key=` query parameter support means the API key appears in server logs and shell history.

**Recommendation**: Prefer `X-Cinder-Key` header for API clients. Query param support is convenient for iOS Shortcuts — leave it but document the tradeoff.
